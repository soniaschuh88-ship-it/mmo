# bKG — NPCs

> AI Game Master · dialogue · quests · behavior layers

---

## 1. Overview

NPCs in bKG are **not scripted**. They are driven by `bifrost-aigm` — the AI Game Master layer — which combines three behavioral layers:

| Layer | System | Description |
|---|---|---|
| Layer 1 | `NpcBehavior` FSM | Reactive state machine — idle, patrol, speak, flee |
| Layer 2 | `DialogueQueue` | LLM-backed dialogue generation with cooldown |
| Layer 3 | `AiContext` | Long-term memory, goals, relationship graph |

```rust
pub struct NpcState {

}
```

---

## 3. Behavior FSM (Layer 1)

Reactive states driven by world events:

```
idle ──(player_near)──► speak ──(done)──► idle
idle ──(timeout)──────► patrol ──────────► idle
patrol ──(threat)──────► flee
speak ──(quest_trigger)► quest_dialogue
```

## 5. Dialogue System

NPC dialogue is generated by `bifrost-aigm::NpcDialogue`:

- Layer 1 FSM triggers dialogue events
- Dialogue queue batches LLM requests with cooldown to control cost
- LLM responses are validated before display
- Short-term memory prevents repetition within a session

```rust
pub struct NpcLlmRequest {
    pub npc_id:           NpcId,
    pub player_context:   PlayerDialogueContext,
    pub world_state:      WorldStateSnapshot,
    pub conversation_history: Vec<DialogueTurn>,
}
```

## 6. NPC HTTP API

```bash
GET  /aigm/npcs                  # list all NPCs in the active zone
GET  /aigm/npcs/:id              # single NPC state
POST /aigm/npcs/:id/speak        # trigger dialogue generation
POST /aigm/tick                  # advance all NPC behavior FSMs
```
